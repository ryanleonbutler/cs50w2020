Git Notes

Overview
Git is a source code, version control system, which keeps track of changes made to source code. It also allows for source code to be synchronized between various contributors, make changes to source code without losing the original copy, and revert back to older versions of the source code.

GitHub is a website that stores Git repositories on the Internet in order to facilitate the collaboration that Git allows for. A repository is a place to keep track of source code and all the changes to that source code.
Commands
git clone <url> --- connects to a repository stored on a server, for example GitHub, and downloads it locally.

$ git clone git@github.com:<user-name>/my-repo;.git
Cloning into 'my-repo'...
remote: Enumerating objects: 96, done.
remote: Counting objects: 100% (96/96), done.
remote: Compressing objects: 100% (67/67), done.
remote: Total 96 (delta 33), reused 87 (delta 24), pack-reused 0
Receiving objects: 100% (96/96), 639.05 KiB | 625.00 KiB/s, done.
Resolving deltas: 100% (33/33), done.

git add <filename(s)> --- adds files to the staging area to be included in the next commit.

$ git add -v test.html
add 'test.html'

git add -A --- adds all files to the staging area to be tracked, without specifying their names.

$ git add -Av
add 'my-test.html'
add 'my-test.txt'

git commit -m "message" --- takes a "snapshot" of the repository and saves it with a message about the changes.

git commit -m "Test commit."
[master a5e3ab8] Test commit.
2 files changed, 0 insertions(+), 0 deletions(-)
create mode 100644 test.html
create mode 100644 test.txt

git status --- prints what is currently going on with the repository, provides and overview.

$ git status
On branch master
Your branch is ahead of 'origin/master' by 2 commits.
 (use "git push" to publish your local commits)

Changes to be committed:
 (use "git reset HEAD <file>..." to unstage)

new file: my-test.html
new file: my-test.txt
